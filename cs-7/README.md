## Вывод
- Первая реализация значительно эффективнее для операции at, особенно при многократных вызовах или работе с большими массивами строк
- Вторая реализация обладает линейной сложностью на каждый доступ, что делает её непригодной для практического использования, если требуется частый произвольный доступ по индексу.
