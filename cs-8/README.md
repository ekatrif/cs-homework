## Запуск парсеров

- **Парсер CSV**: npm run parse-csv
- **Парсер JSON**: npm run parse-json

## Результаты парсинга CSV

- **Общее время**: 373.72 ms
- **Время до первого байта (latency)**: 2.82 ms
- **Heap size (память)**: 20.55 MB
- **Размер до сжатия**: 10.54 MB
- **Размер после сжатия Gzip**: 1.24 MB

## Результаты парсинга JSON

- **Общее время**: 133.37 ms
- **Время до первого байта (latency)**: 1.75 ms
- **Heap size (память)**: 54.38 MB
- **Размер до сжатия**: 10.70 MB
- **Размер после сжатия Gzip**: 1.24 MB

## Выводы
1. Json.parse() работает быстрее парсинга CSV (скорее всего причина этому - хорошая оптимизация в V8)
2. JSON потребляет больше памяти
3. GZIP одинаково хорошо сжимает оба формата
