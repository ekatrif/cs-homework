## Ключевые выводы

### 🔥 Самый важный результат
**На больших массивах (100k) Holey массивы значительно быстрее Regular для `push` — в 2.25 раза без JIT и в 2.1 раза с JIT.** Это противоречит интуиции и предыдущим тестам.

### 📊 Другие важные наблюдения

1. **`push` — аномальное поведение:**
   - На 100k: Holey лидирует с огромным отрывом (0.92 против 2.07)
   - На 1000-10000: без JIT Holey быстрее, с JIT — Regular быстрее
   - JIT радикально меняет расклад сил

2. **`pop` — Regular стабильно быстрее на больших размерах:**
   - На 100k преимущество ~46-47% независимо от JIT

3. **`shift` — равновесие на средних и больших размерах:**
   - На 10000 — идеальное равенство
   - На 100000 — оба очень медленные (~760 мс)

4. **`unshift` — JIT даёт существенное ускорение:**
   - На 100 — ускорение на 12-15%
   - На 10000 — ускорение на 8-9%
   - Но операция остаётся крайне медленной (~900-2500 мс)

5. **JIT-прогрев даёт противоречивый эффект:**
   - Сильно ускоряет `unshift` (до 15%)
   - Улучшает `shift` на больших размерах (~3%)
   - Для `push` может как ускорить, так и замедлить в зависимости от размера

### ⚡ Рекомендации по использованию

| Сценарий | Рекомендуемый тип массива |
|----------|--------------------------|
| Много операций `push` на больших данных (100k+) | **Holey** |
| Много операций `pop` любого размера | **Regular** |
| Много операций `shift` | Оба плохи, лучше использовать другую структуру |
| Много операций `unshift` | Оба очень медленные, использовать deque/связный список |
| Непредсказуемый размер, но мало операций | Без разницы |

### 🎯 Главный вывод
**Вопреки распространённому мнению, Holey (разреженные) массивы могут быть значительно быстрее Regular для `push` на больших объёмах данных (до 2.25×).** Это важное наблюдение, которое требует дальнейшего исследования причин такого поведения в движке JavaScript.
