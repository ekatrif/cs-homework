## Другие важные наблюдения

### 1. `push` — Regular стабильно быстрее после JIT
- На 100: Regular быстрее в **6.2 раза** (0.26 против 1.61)
- На 1000: Regular быстрее в **2.1 раза**
- На 10000: Regular быстрее на **44%**
- На 100000: Regular быстрее в **3.3 раза**

### 2. `pop` — небольшое преимущество Regular
- На 100000 с JIT: Regular быстрее в 2 раза (0.47 против 0.95)

### 3. `unshift` — остаётся очень медленным для обоих типов
- Даже с JIT: ~2300-2600 мс на 100000
- Разница между Holey и Regular минимальна

### 4. Эффективность JIT для Regular массива
JIT невероятно эффективно оптимизирует Regular массив для:
- `push` — ускорение в 2-4 раза
- `shift` на больших размерах — **ускорение в 1700+ раз**

## Главные выводы

1. **Для production-кода с JIT (реальная среда) — используйте Regular массивы.** Они стабильно быстрее или хотя бы не медленнее для всех операций, а для `shift` на больших данных — несравнимо быстрее.

2. **Holey массивы могут быть опасны для производительности** после прогрева JIT — в некоторых случаях (например, `shift` на 100000) они в тысячи раз медленнее.

3. **JIT-компилятор не одинаково оптимизирует Holey и Regular массивы** — Regular получает агрессивные оптимизации, которые недоступны для разреженных массивов.

4. **`shift` на больших Regular массивах с JIT — это O(1) операция?** Результат 0.44 мс для 100000 элементов предполагает, что движок использует оптимизации, меняющие представление массива.

5. **`unshift` остаётся узким местом для обоих типов** — для частых вставок в начало используйте другие структуры данных (связный список, deque).
